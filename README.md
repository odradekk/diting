# diting

diting is a multi-source aggregated search CLI that answers a technical question by querying up to 8 heterogeneous sources, fetching the highest-quality results, and producing a cited answer optimised for consumption by another LLM. It is a complete Go rewrite of a Python v1, ships as a single static binary, and is strictly BYOK — no credentials are bundled or transmitted to any diting-operated endpoint.

## Install

**One-line (Linux/macOS):**

```sh
curl -fsSL https://raw.githubusercontent.com/odradekk/diting/main/install.sh | sh
```

**From source:**

```sh
go install github.com/odradekk/diting/cmd/diting@latest
```

## Quick start

```sh
# Generate ~/.config/diting/config.yaml
diting init

# Set at least one LLM key
export ANTHROPIC_API_KEY=your-key-here

# Verify the environment
diting doctor

# Run a search
diting search "how do I mount a projected service account token in Kubernetes?"
```

## BYOK

Every external API is BYOK. diting reads credentials from environment variables (and from `~/.config/diting/config.yaml` if generated by `diting init`).

| Variable | Required | Purpose |
|---|---|---|
| `ANTHROPIC_API_KEY` | Yes (if using Anthropic) | Anthropic Claude models |
| `OPENAI_API_KEY` | Yes (if using OpenAI or compatible) | OpenAI or OpenAI-compatible models |
| `OPENAI_BASE_URL` | No | Override base URL for OpenAI-compatible providers (e.g. MiniMax: `https://api.minimaxi.com/v1`) |
| `OPENAI_MODEL` | No | Model override for the OpenAI-compatible endpoint |
| `BRAVE_API_KEY` | No | Enables Brave Search module (higher quality than scraping) |
| `SERP_API_KEY` | No | Enables SerpAPI module (Google results; ~100 free queries/month) |
| `TAVILY_API_KEY` | No | Enables Tavily extract layer (paid last-resort fetch fallback) |
| `GITHUB_TOKEN` | No | Lifts GitHub Search from 10 to 30 requests/minute |
| `JINA_API_KEY` | No | Raises Jina fetch layer rate limit (anonymous access still works) |

At least one LLM key (`ANTHROPIC_API_KEY` or `OPENAI_API_KEY`) is required for the `search` subcommand to produce answers. The others enable optional capability boosts. For DeepSeek Chat, set `OPENAI_API_KEY` to your DeepSeek key and `OPENAI_BASE_URL=https://api.deepseek.com`.

## Commands

| Command | Description |
|---|---|
| `diting search <question>` | Full pipeline: plan, search, fetch, synthesise answer |
| `diting fetch <url>` | Fetch and extract content from a URL |
| `diting config <show\|path\|validate>` | Inspect or validate the config file |
| `diting init` | Interactive config generator |
| `diting doctor` | Environment health check (keys, binary deps, connectivity) |
| `diting bench <run\|report>` | Run the benchmark suite or view the latest report |
| `diting version` | Print version string |

For flag reference: `diting search --help`, `diting fetch --help`.
